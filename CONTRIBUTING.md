# Contributing to My Portfolio

Thank you for your interest in contributing! This document provides guidelines and instructions for contributing.

## Code of Conduct

- Be respectful and constructive
- Welcome diversity of opinions
- Focus on the project goals

## How to Contribute

### Reporting Bugs

1. Check existing issues to avoid duplicates
2. Provide:
   - Clear title and description
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots if applicable
   - System information

### Suggesting Enhancements

1. Use clear, descriptive title
2. Explain the enhancement and benefits
3. Provide examples if possible
4. Link related issues

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Commit with clear messages
5. Push to your fork
6. Open a Pull Request with:
   - Clear description of changes
   - Reference to any related issues
   - Screenshots for UI changes

## Development Setup

```bash
git clone https://github.com/mdivyalalitha29/my-portfolio.git
cd my-portfolio
npm install
npm start
```

## Code Style

- Use meaningful variable names
- Add comments for complex logic
- Keep components focused and reusable
- Follow existing code patterns

## Testing

```bash
npm test
npm run build
```

## Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit to 72 characters
- Reference issues and pull requests liberally after the title

## Questions?

Feel free to open an issue with the `question` label.
